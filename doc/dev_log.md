# 开发日志 — 任务二：桌面小云台自瞄

## 2026-07-22

### 相机标定
- **做了什么**: 用 `gen_calib_pattern.py` 生成 11×8 棋盘格（17mm），屏幕显示后用 `capture_desktop` 采集 25 张标定图像，`calibrate_checkerboard` 计算内参
- **结果**: fx=1721.9, fy=1720.3, cx=638.6, cy=526.5，重投影误差 0.14px
- **坑**: 一开始标定板检测失败（9×6 vs 11×8 不匹配），加 hysteresis 防抖后解决

### 串口通讯协议
- **做了什么**: 分析下位机协议（0x5A 0xA5 帧头，43 字节），实现 `GimbalToVision` 结构体解析
- **结果**: "First q received"，云台四元数数据正常接收
- **坑**: 
  - 帧尾 0x7F 0xFE 校验失败 → 固件实际不发帧尾，去掉校验
  - 数据 sanity check 失败（yaw=-15° 被当弧度拒绝）→ 固件发的是度，不是弧度
  - InferRequest 移动语义问题 → ThreadSafeQueue 加了 `push(T&&)` 重载

### YOLO 模型
- **做了什么**: 
  1. 拍 110 张矿石照片，用 label_tool 点击标注
  2. 用 yolo11n 训练 200 轮（CPU 约 15 分钟）
  3. 导出 OpenVINO IR，部署到 `assets/`
- **结果**: 置信度极低（最高 0.07），模型基本不可用
- **原因分析**: 88 张训练集太少，标签框固定 0.08 尺寸不准
- **教训**: 小数据集用 YOLO 效果差，改用传统方法

### 跟踪器（SimpleTracker）
- **做了什么**: 三版迭代
  1. **帧差法** → 检测到目标但云台自激振荡（检测到云台自身运动）
  2. **自适应阈值** → 锁住背景不动
  3. **滑动平均背景差法**（α=0.02）→ 只检测运动目标，不受云台影响
- **结果**: 云台不抽风了，能跟踪运动目标
- **控制参数**: KP_YAW=0.08, KP_PITCH=0.05, DEADBAND=10px
- **待改进**: 跟踪响应偏慢，P 参数需进一步调优
